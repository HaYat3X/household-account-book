# 家計管理Webアプリ 要件定義書

**作成日**: 2026-05-06  
**対象**: はやて・彼女（2人用）  
**ステータス**: Draft

---

## 1. 概要

2人用の家計管理Webアプリ。レシート読み取り・共同予算管理・立替精算を中心とした機能を提供する。

---

## 2. ユーザー・認証

- Googleログイン（NextAuth.js）
- 2アカウントでデータ共有（カップル単位のワークスペース）
- 招待制でカップルを紐付ける

---

## 3. 予算設定

月次予算の合計は2人で24万円（各自12万円を共同口座に入金）。

| カテゴリ | 月次予算 |
|---|---|
| 家賃 | ¥100,000 |
| 光熱費 | ¥20,000 |
| 食費 | ¥60,000 |
| 日用品 | ¥20,000 |
| 貯金 | ¥40,000 |
| その他 | 例外的な支出 |

カテゴリの細分化は運用後に決定する。

---

## 4. 機能一覧

### 4.1 コア機能

| 機能 | 詳細 |
|---|---|
| レシート読み取り | 画像アップロード → Claude Vision OCR → 金額・店舗・商品リスト抽出 |
| 商品→カテゴリ分離 | 1枚のレシートで複数カテゴリに振り分け（手動修正可） |
| 手動入力 | レシートなしでも支出を直接登録可能 |
| カテゴリ自動推論 | 店舗名・商品名からカテゴリをClaude APIで推論 |
| 立替管理 | 立替登録 → アプリ内通知 → 返金完了フラグ |
| 月次ダッシュボード | 予算vs実績・カテゴリ別グラフ |
| 予算設定 | カテゴリ別月次予算の管理 |

### 4.2 立替フロー

```
はやてくんが立て替え
  ↓
アプリに登録（金額・カテゴリ・メモ）
  ↓
「請求する」ボタン押下 → 彼女にアプリ内通知
  ↓
彼女が実際に返金（口座振込など）
  ↓
はやてくん or 彼女が「返金完了」フラグをON
```

> **未決**: 通知タイミングを登録即時にするか、「請求する」ボタン押下時にするか。金額修正後に請求するほうが安全なため、ボタン押下式を推奨。

### 4.3 レシート対象店舗

- スーパー
- コンビニ
- ドラッグストア
- 外食

### 4.4 運用ルール

- 共同の外食・食料品など → 食費カテゴリで記録
- 個人のおやつ・個人的な買い物 → アプリに記録しない

### 4.5 将来構想（v2）

- PWA化・プッシュ通知
- カテゴリ細分化（運用後に確定）
- 月次レポート

## 6. 技術スタック

| 領域 | 採用技術 | 理由 |
|---|---|---|
| フレームワーク | Next.js (App Router) | 既存スキル |
| 認証 | NextAuth.js | Google OAuth・無料 |
| DB | Supabase (PostgreSQL) | 無料枠で十分・RLS・Storage一体 |
| ORM | Prisma | 型安全・Supabase相性◎ |
| OCR | Claude API (Vision) | レシート解析精度・プロンプト制御しやすい |
| UI | Tailwind | 既存スキル |
| デプロイ | Vercel | 無料枠・Next.js相性◎ |
| 画像保存 | Supabase Storage | 無料枠500MB |

### 無料枠の目安

- Supabase: DB 500MB・Storage 1GB
- Vercel: 帯域 100GB/月
- Claude API: レシート読み取り1回あたり約0.3〜0.5円（従量課金のみ）

2人運用であれば当分無料枠で回る。
